System Overview

How the System Works

The checker resolves copyright ownership for any Spotify track by chaining Spotify metadata → credit parsing → registry lookups → audio fingerprinting → AI synthesis. Each layer narrows uncertainty and increases confidence in the final verdict.

Data Pipeline Steps

Step-by-Step Flow

Each step enriches the copyright picture. Steps 1–3 are always run. Steps 4–6 are triggered only when earlier steps return incomplete or ambiguous data.

01
Input & Auth

Spotify Track Resolution

Accept track URL, URI, or ISRC. Exchange Spotify client credentials for an access token. Fetch full track object including ISRC, album details, and all artist IDs.

  • GET /tracks/{id} — full metadata
  • GET /albums/{id} — label, release date, copyright field
  • Extract album.copyrights[] array (℗ + ©)
  • Extract external_ids.isrc for registry matching
02
Credits Parsing

Spotify Credits Endpoint

The internal Spotify credits endpoint (via partner API or web scraping the track page) returns structured credit data not available in the public API.

  • Songwriters / Composers (mechanical rights)
  • Record producers (sound recording rights)
  • Publishers (sync & print rights)
  • Recording label (master rights)
  • Performing rights org (PRO affiliation)
03
Registry Search

PRO & Copyright Registry Lookups

Use writer/publisher names extracted from credits to query performing rights organizations and copyright registries. Run searches in parallel.

  • ASCAP ACE database (title + writer search)
  • BMI Repertoire search
  • SESAC repertory lookup
  • US Copyright Office (copyright.gov/public-records)
  • PRS for Music (UK), SOCAN (Canada)
04
Cross-Reference

MusicBrainz + Wikidata Enrichment

Query MusicBrainz for MBID, release group, label relations. Cross-reference against Wikidata for disambiguation and dispute history.

  • MusicBrainz ISRC lookup → canonical recording
  • Label relation → identify distributor vs. rights holder
  • Wikidata: ownership transfers, mergers, splits
  • Discogs label catalog for indie verification
05
Audio Fingerprint

Acoustic Fingerprint Verification

Fingerprint the 30-second Spotify preview URL against AcoustID and AudD databases to confirm identity and surface alternate registrations.

  • Download preview_url (30s MP3, no auth needed)
  • Generate AcoustID fingerprint via fpcalc
  • Query AcoustID API → MBID confirmation
  • AudD API → song ID + Apple Music / Deezer cross-ref
  • Detect covers, samples, or re-recordings
06
AI Synthesis

Multi-Agent Reasoning + Verdict

The orchestrator AI agent receives all collected data, resolves conflicts, identifies gaps, calls web search for missing info, and produces a structured copyright verdict.

  • Tool: web_search for dispute / lawsuit history
  • Tool: fetch PRO pages if scraping needed
  • Identify master vs. composition rights holders
  • Classify: Original / Cover / Sample / Public Domain
  • Output: confidence score, risk level, licensing path

AI Agent Architecture

Multi-Agent Design

A chain of specialized AI agents — each with distinct tools and responsibilities — produces a copyright verdict that no single API could provide alone.

Orchestrator Agent
Claude Sonnet / GPT-4o · Main controller

Receives the Spotify track ID and manages the full pipeline. Decides which tools to call, in which order, and when to stop. Synthesizes all sub-agent outputs into a final copyright report.

spotify_lookup credits_fetch dispatch_agents aggregate_results web_search
Credits Parser Agent
LLM + regex · Structured extraction

Takes raw HTML or JSON from the Spotify credits endpoint / page and extracts structured entities: writers, composers, producers, publishers, PRO affiliations. Handles inconsistent formatting.

html_parse entity_extract normalize_names dedup_credits
Registry Lookup Agent
Tool-use · Parallel API calls

Queries ASCAP, BMI, SESAC, Copyright Office, and international PROs in parallel using writer/title data. Normalizes responses into a unified rights object with source attribution.

ascap_search bmi_search copyright_office intl_pro_search
Discrepancy Resolver
Reasoning LLM · Conflict detection

Compares data across all sources, flags conflicts (e.g. different publishers in ASCAP vs BMI), and attempts resolution via additional web searches, Wikipedia, or music news databases.

conflict_detect web_search news_search explain_gap
Sample Detector
Audio analysis · WhoSampled API

Checks if the track contains samples that require additional clearance. Queries WhoSampled.com (scrape/unofficial API) and cross-refs with known sample disputes. Also checks interpolations.

whosampled_lookup interpolation_check derivative_rights
Risk Assessor Agent
Classifier · Final verdict writer

Takes the synthesized rights object and outputs: copyright type, rights holder(s) for master + composition, risk level for use, recommended licensing path, and confidence percentage.

classify_risk license_path confidence_score render_report

Implementation Snippets

Code Reference

Key implementation patterns for each stage of the pipeline.

STEP 1 · SPOTIFY TRACK FETCH

// Fetch track metadata + album copyright const token = await getSpotifyToken(CLIENT_ID, CLIENT_SECRET); const track = await fetch(`https://api.spotify.com/v1/tracks/${trackId}`, { headers: { Authorization: `Bearer ${token}` } }).then(r => r.json()); const album = await fetch(`https://api.spotify.com/v1/albums/${track.album.id}`, { headers: { Authorization: `Bearer ${token}` } }).then(r => r.json()); // Extract key copyright fields const data = { isrc: track.external_ids.isrc, previewUrl: track.preview_url, // 30s MP3 for fingerprinting copyrights: album.copyrights, // [{type:"P", text:"℗ 2019 Republic"}, ...] label: album.label, releaseDate: album.release_date };

STEP 2 · SPOTIFY CREDITS PAGE PARSE

// Spotify credits page (no auth needed for public tracks) const creditsHtml = await fetch( `https://open.spotify.com/track/${trackId}`, { headers: { 'User-Agent': 'Mozilla/5.0' } } ).then(r => r.text()); // Extract __NEXT_DATA__ JSON which contains credits const match = creditsHtml.match(/<script id="__NEXT_DATA__"[^>]*>([^<]+)/); const nextData = JSON.parse(match[1]); const credits = nextData?.props?.pageProps?.track?.credits; // Pass raw credits to AI parsing agent const structured = await creditsParserAgent(credits); // → { writers: [], producers: [], publishers: [], proAffiliations: [] }

STEP 3 · MUSICBRAINZ ISRC LOOKUP

const mb = await fetch( `https://musicbrainz.org/ws/2/isrc/${isrc}?inc=releases+labels+artist-credits&fmt=json`, { headers: { 'User-Agent': 'CopyrightChecker/1.0 (you@email.com)' } } ).then(r => r.json()); const recording = mb.recordings[0]; const label = recording?.releases?.[0]?.["label-info"]?.[0]?.label?.name;
